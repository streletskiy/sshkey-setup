# tnnl-key

Интерактивный bash-скрипт для Ubuntu.

Что делает:
- добавляет публичный SSH-ключ в `authorized_keys`
- включает вход по SSH-ключу через `PubkeyAuthentication yes`
- отключает вход по паролю через `PasswordAuthentication no`

## Как запускать

Скрипт специально читает ввод из `/dev/tty`, поэтому его удобно запускать через `curl`.

Готовая команда для копирования:

```bash
curl -fsSL https://raw.githubusercontent.com/streletskiy/sshkey-setup/785afd5/setup-ssh.sh | sudo bash -s --
```

Это зафиксированная версия без риска поймать кеш GitHub Raw сразу после обновления. Если нужен именно `main`, можно заменить `785afd5` на `main`, но ветка может обновляться с небольшой задержкой.

Если файл уже скачан локально:

```bash
sudo bash setup-ssh.sh
```

## Меню

После запуска будет меню:

1. Добавить публичный SSH-ключ
2. Отключить вход по паролю

### Пункт 1

- Скрипт спросит пользователя, для которого нужно добавить ключ
- Затем попросит вставить публичный SSH-ключ
- Ключ попадёт в `~/.ssh/authorized_keys`
- Если такой ключ уже есть, дубликат не добавляется

### Пункт 2

- Скрипт попросит подтверждение словом `DISABLE`
- Перед изменением создаётся резервная копия `/etc/ssh/sshd_config`
- Конфиг проверяется через `sshd -t` до применения и после

## Корнер-кейсы, которые учтены

- запуск без `root` или `sudo` завершается ошибкой
- запуск без TTY завершается ошибкой
- невалидный SSH public key отклоняется
- симлинки для `~/.ssh` и `authorized_keys` не принимаются
- повторное добавление того же ключа не создаёт дубль
- `sshd_config` обновляется через управляемый блок и валидируется перед reload
- если автоматический reload не сработает, скрипт покажет ручную команду

## Важно

Отключение пароля может отрезать доступ к серверу, если ключевой вход не проверен. Сначала проверь вход в новой SSH-сессии, и только потом отключай пароль.
